Creating a QiType

To insert data into Qi, you must first create a QiType that will enforce what type of data is written to a stream. For this sample, we will be writing events of the WaveData type discussed in the README. Press the button below to create the WaveData QiType.

Retrieving events from Qi

There are several ways to read data back from Qi. Using GetRangeValues, we can start at index "3" of our wave data and get the next 50 events. We didn't actually write a value with an index of 3, but since we use QiBoundaryType.ExactOrCalculated in our request, we get an interpolated value at 3 if the "Retrieve Events" button is pressed again.

Using QiStreamBehaviors

The default behavior for a QiStream is Continuous, which will allow interpolation when a requested index isn't present. In order to change this to only give us exact values that were written, we can create a behavior with Discrete mode and update the stream to use it. Once the stream is updated, we should no longer see a value at 3.
